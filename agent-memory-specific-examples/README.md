# Agent Memory Specific Examples

This folder contains more specific examples of memory-like files beyond the basic daily/topic/current-state categories.

## Included
- `example-session-memory.md` — representative session/current-state memory file from the global `memory/` directory
- `example-project-memory.md` — representative project-level current-state / resume spine
- `example-project-closeout.md` — representative project-level closeout / execution boundary file

## Category explanation
### 1. Session memory
Purpose:
- dense state capture for one active session or work window
- useful when a conversation becomes large and a resume spine is needed

Example:
- `example-session-memory.md`

### 2. Project memory-like file
Purpose:
- project-local resume anchor
- current status, reading order, boundaries, next steps
- not global memory, but functions similarly inside one repo

Example:
- `example-project-memory.md`

### 3. Project closeout / phase-boundary file
Purpose:
- codify what is done, what remains, what should stop, and what counts as closeout
- this is more execution governance than raw memory, but often plays a memory-like role for resuming work correctly

Example:
- `example-project-closeout.md`

## Important distinction
These project files are not part of the agent's global memory layer.
They are project-internal state artifacts that reduce resume cost for one repo/workstream.
