# Agent Memory Examples

This folder contains one representative file for each major memory category.

## Included examples
- `example-daily-log.md` — representative daily memory file
- `example-current-state.md` — representative session/current-state memory file
- `example-topic-log.md` — representative topic-specific memory file
- `INDEX.md` — explains the broader memory directory structure

## Category mapping
### 1. Daily log
Purpose:
- chronological summary of durable events, decisions, and preference updates

Representative file:
- `example-daily-log.md`

### 2. Current-state / session-state file
Purpose:
- dense handoff/resume context for an active workstream or session

Representative file:
- `example-current-state.md`

### 3. Topic-specific log
Purpose:
- detailed thread for one focused problem or implementation track

Representative file:
- `example-topic-log.md`

## Why only one of each
These examples are meant to teach the pattern, not dump every variant.
